# Автоматизация разработки с использованием Make

## Описание

В данном проекте реализована автоматизация основных этапов разработки Python-приложения с помощью Makefile.  
Make используется как единая точка входа для управления окружением, зависимостями и проверками качества кода.

Цель работы — показать, как организовать воспроизводимый процесс разработки.

---

## Работа с окружением

Используется локальное виртуальное окружение `.venv`.

Вместо `source .venv/bin/activate` используется stateless-подход:
- `.venv/bin/python`
- `.venv/bin/pip`

Это связано с тем, что Make выполняет каждую строку в отдельном shell.

---

## Основные команды

Создание окружения:
make venv

Установка зависимостей:
make install-dev

Запуск:
make run

---

## Проверки

Проверка зависимостей:
make check-requirements

Проверка типов:
make typecheck

Форматирование:
make format

Проверка стиля:
make lint

Тесты:
make test

---

## Композиция

make check

Объединяет все проверки.

---

## Вывод

Makefile используется как единая точка управления процессом разработки.
